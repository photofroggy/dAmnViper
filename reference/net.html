2. net module¶

This module provides classes used to actually connect to dAmn using Twisted. The ConnectionFactory starts connections and handles disconnects. The IOProtocol handles basic IO operations on the connection, but delegates most of the processing to an instance of the dAmnSock class.

2.1 ConnectionFactory - Connection management¶

class dAmnViper.net.ConnectionFactory(client, stdout=None, debug=None)¶

Connection management.

Instances of this class are used by dAmnSock to connect to dAmn.

The object returns instances of IOProtocol when a connection is made and handles disconnects and connection failures.

This object should not be used directly by applications using dAmnViper.

buildProtocol(addr)¶

Called by twisted when a protocol is needed.

Twisted calls this method when a connection can be established with the server. This method creates an instance of the IOProtocol class. This instance is given to the dAmnSock object that the factory belongs to, and returned to twisted.

clientConnectionFailed(connector, reason)¶

Called by twisted when we fail to connect properly.

The behaviour of this method is similar to the clientConnectionLost method.

clientConnectionLost(connector, reason)¶

Called by twisted when a connection is lost.

Displays a message notifying the connection loss and tells the dAmnSock object that there is no longer a connection open.

startedConnecting(connector)¶

Called by twisted when we start to connect.

All this method really does is display a message and set the connecting flag to True.

2.2. IOProtocol - Input and Output piping¶

class dAmnViper.net.IOProtocol(conn, client, stdout=None, debug=None)¶

Protocol layer for the dAmn connection.

Instances of this class are used to directly communicate with the connection to dAmn via twisted, and gives any data received to the dAmnSock instance being used for this connection.

This object should not be used directly by applications using dAmnViper.

connectionMade()¶

Called by twisted when we have connected.

This method simply sends a handshake to the dAmn server, and sets the client’s shaking flag to True, indicating that a handshake packet has been sent.

dataReceived(data)¶

Called by twisted when data is received.

The data received is added to out buffer. If there are any full packets in the buffer, these packets are sent to the dAmnSock instance to be parsed properly.

Any event handling relating to specific packet is done in dAmnSock instance.

send_packet(data)¶

A wrapper function for sending packets to the server.

Here, a null character (\0) is appended to the given data, and we return the number of characters we have tried send to the server.
